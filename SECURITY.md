# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within FeedbackHub, please send an email to [security@feedbackhub.example.com](mailto:security@feedbackhub.example.com) instead of using the public issue tracker.

Please include the following details in your report:
- A description of the vulnerability and its impact
- Steps to reproduce or proof-of-concept
- Versions affected
- Any known mitigations

All security vulnerabilities will be promptly addressed. You will be credited for your discovery unless you request otherwise.

## Security Measures

We take security seriously and have implemented the following measures:
- Regular dependency updates
- Input validation and sanitization
- Secure HTTP headers
- Protection against common web vulnerabilities (XSS, CSRF, etc.)

## Best Practices for Users

To ensure the security of your FeedbackHub installation:
- Keep your installation up to date with the latest version
- Use strong, unique passwords
- Regularly backup your data
- Restrict access to the administration interface
- Monitor logs for suspicious activity
